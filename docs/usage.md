# 使い方ガイド

## 画面構成

```
┌─────────────────────────────────────────────────────────┐
│  🎵 Local Music Player                  [Syncthing: off]│
├─────────────────────────────────────────────────────────┤
│  [Queue]          [Playlists]          [Library]        │  ← モバイルのみ
├──────────────┬──────────────────┬────────────────────────┤
│   Queue      │   Playlists      │      Library           │
│  ← 左パネル  │  ← 中央パネル   │    ← 右パネル →        │
└──────────────┴──────────────────┴────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│  [サムネ]  タイトル / アーティスト     ⏮ ▶ ⏭           │  ← 再生バー
│            ────●──────────────  0:42 / 3:33            │
└─────────────────────────────────────────────────────────┘
```

デスクトップでは 3 カラムが並んで表示されます。モバイルでは上部のタブで切り替えます。

---

## 1. 音楽をダウンロードする（Queue パネル）

### 手順

1. 左パネル（Queue）上部の入力欄に YouTube URL を貼り付ける
2. フォーマット（MP3 / FLAC / AAC / OGG）と品質（192 / 320 / Best kbps）を選択
3. **「+ Add & Download」** ボタンをクリック

登録が完了すると **Registered URLs** に追加され、バックグラウンドで解決・ダウンロードが始まります。

### 対応 URL

| 種別 | URL 例 |
|------|--------|
| 動画 1 件 | `https://www.youtube.com/watch?v=dQw4w9WgXcQ` |
| プレイリスト | `https://www.youtube.com/playlist?list=PLxxxx` |
| チャンネル | `https://www.youtube.com/@ChannelName` |

プレイリスト・チャンネルを登録すると、含まれる全動画がキューに追加されます。

### ダウンロード進捗の確認

**Download Queue** セクションにリアルタイムで進捗バーが表示されます。

| 表示 | 意味 |
|------|------|
| `pending` | キュー待ち |
| `downloading` + バー | ダウンロード中 |
| `complete` | 完了（Library に追加済み） |
| `failed` + ↺ ボタン | 失敗（↺ でリトライ） |

### 登録 URL の削除

**Registered URLs** の各行の **✕** ボタンをクリックします。ダウンロード済みファイルを残すかどうかは確認なしに設定のみ削除されます。

---

## 2. 音楽を再生する（Library パネル）

1. 右パネル（Library）で曲を探す
2. 検索ボックスにタイトルまたはアーティスト名を入力してフィルタ
3. トラックカードをクリックすると画面下部の **プレイヤーバー** で再生開始

### プレイヤー操作

| 操作 | 説明 |
|------|------|
| **▶ / ⏸** | 再生 / 一時停止 |
| **⏮** | 前の曲（同じ検索結果内） |
| **⏭** | 次の曲（同じ検索結果内） |
| シークバー | 任意の位置にドラッグ |

> **ヒント**: トラック一覧でクリックすると、表示されている全トラックがプレイリストとしてプレイヤーにセットされます。最後の曲が終わると自動停止します。

### トラックのダウンロード

各トラックカードの **↓** ボタンで音声ファイルをブラウザにダウンロードできます。

### トラックの削除

各トラックカードの **🗑** ボタンをクリックすると、DB レコードと音声ファイル・サムネイルをすべて削除します（確認ダイアログあり）。

## 3. YouTube プレイリストを自動同期する（Playlists パネル）

YouTube アカウントのプレイリストを定期的に自動同期します。

### 初回設定（OAuth 認証）

1. 中央パネル（Playlists）の **「YouTubeアカウントに接続」** をクリック
2. Google のログイン画面でアカウントを選択し、アクセスを許可
3. アプリに戻り「YouTube接続済み」と表示されたら完了

!!! note "事前準備"
    OAuth 認証を使うには `.env` に `YOUTUBE_CLIENT_ID` と `YOUTUBE_CLIENT_SECRET` を設定する必要があります。  
    設定方法は [YouTube OAuth2 のセットアップ](deployment/configuration.md#youtube-oauth2-のセットアップ) を参照してください。

### プレイリストの同期追加

1. 「アカウントのプレイリスト」に自分のプレイリストが表示される
2. 同期したいプレイリストの **「+ 同期追加」** をクリック
3. 「同期中のプレイリスト」に追加され、ダウンロードが自動開始

!!! warning "フォーマット選択"
    現時点では同期追加時のフォーマット・品質はデフォルト（MP3 / 192 kbps）固定です。  
    フォーマット選択機能は [今後の課題](future-work.md#プレイリスト同期追加時にフォーマット選択ができない) として記載されています。

### 同期設定の管理

「同期中のプレイリスト」カードの各ボタン:

| ボタン | 動作 |
|--------|------|
| **同期** | 今すぐ同期タスクを実行 |
| **一時停止 / 再開** | 自動同期の有効・無効を切り替え |
| **削除** | 設定を削除（ファイルも削除するか選択可） |
| **トラック一覧を表示** | 同期済みトラックの一覧を展開 |

### 自動同期間隔の設定

「自動同期間隔 / YouTubeプレイリスト」セレクターで間隔を変更します。

| 選択肢 | 動作 |
|--------|------|
| 手動のみ | 自動同期しない（「同期」ボタンで手動実行） |
| 15分〜24時間 | 指定間隔で自動同期 |

## 4. Syncthing でモバイルに同期する

`downloads/` フォルダを Syncthing で共有すると、スマートフォンに音楽ファイルを自動転送できます。

### セットアップ手順

1. PC と スマートフォンに [Syncthing](https://syncthing.net/) をインストール
2. PC の Syncthing GUI（デフォルト: `http://localhost:8384`）を開く
3. 設定 → API キーをコピーして `.env` の `SYNCTHING_API_KEY` に設定
4. Syncthing で `downloads/` フォルダを共有フォルダとして追加
5. スマートフォンの Syncthing でデバイスを追加し、フォルダを承認

同期が設定されると、画面右上の **Syncthing バッジ** に状態が表示されます。

| バッジ表示 | 意味 |
|-----------|------|
| `Syncthing: off` | 未設定または接続不可 |
| `Syncing…` | 同期中 |
| `Synced 100%` | 同期完了 |

---

## 5. 自動同期間隔を設定する（Queue パネル）

登録した URL（プレイリスト・チャンネル）の定期チェック間隔を設定します。

**Queue パネル下部の「自動同期間隔 / URLキュー」** セレクターで変更します。

- **手動のみ**: URL を再登録するか、ワーカーを再起動しない限り自動チェックしない
- **15分〜24時間**: 設定した間隔で全 URL を再チェックし、新着動画を自動ダウンロード

## 6. 管理操作

### DB とファイルシステムの同期

ファイルをディレクトリから手動削除した場合など、DB に存在するがファイルがないトラックを一括削除します。

```bash
curl -X POST http://localhost:8000/api/v1/admin/rescan
# → {"removed": 3}
```

### ヘルスチェック

サービスの状態を確認します。

```bash
curl http://localhost:8000/api/v1/health
# → {"status":"ok","redis_connected":true,"db_ok":true,"worker_active":true}
```

| フィールド | 説明 |
|-----------|------|
| `status` | `ok` または `degraded`（Redis/DB いずれかが障害） |
| `redis_connected` | Redis への接続状態 |
| `db_ok` | DB への接続状態 |
| `worker_active` | Celery ワーカーが応答しているか |

## 7. よくある問題

### URL を登録してもダウンロードが始まらない

- Celery ワーカーが起動していることを確認してください
- `GET /api/v1/health` で `worker_active: false` の場合はワーカーを再起動します

```bash
# Docker の場合
docker compose restart worker
```

### ダウンロードが `failed` になる

- `Download Queue` の `failed` ジョブの **↺** ボタンでリトライできます
- 動画が非公開・削除済みの場合は再試行しても失敗します
- yt-dlp のバージョンが古い場合は更新が必要です

### 音楽が再生できない

- トラックカードをクリックして再生バーが表示されない場合、音声ファイルが存在しない可能性があります
- `POST /api/v1/admin/rescan` を実行して孤立レコードを削除してください

### YouTube 認証が失敗する

- `.env` の `YOUTUBE_CLIENT_ID`・`YOUTUBE_CLIENT_SECRET` が正しく設定されているか確認してください
- Google Cloud Console でリダイレクト URI が登録されているか確認してください

### Syncthing バッジが「off」のまま

- `SYNCTHING_API_KEY` が設定されているか確認してください
- Syncthing が起動しているか、`SYNCTHING_URL` が正しいか確認してください
